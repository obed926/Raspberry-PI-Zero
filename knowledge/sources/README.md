# Source Dossiers

Use this lane for ecosystem knowledge that informs decisions but is not an active default runtime tool.

Examples:

1. Framework ecosystem maps
2. Open-source project/plugin inventories
3. Comparative architecture notes
4. First-party operating model guidance (for process design, not runtime defaults)

## Rules

1. Keep source dossiers separate from `tools-shortlist.md`.
2. Promote to default tools only through a logged decision in `docs/DECISIONS.md` and matching registry updates.
3. Each dossier should include: source link, verified claims, status (`watchlist`, `active-reference`, `pending-verification`, `archived-low-signal`), and revisit triggers.
