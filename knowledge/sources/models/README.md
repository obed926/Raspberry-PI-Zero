# Model Source Dossiers

Use this lane for model-family intelligence that should be reusable across projects.

## What belongs here

1. model family overviews (versions, tiers, context windows, licenses)
2. strengths/weaknesses summaries with source links
3. deployment and integration constraints
4. evaluation templates and decision rules for optional adoption

## What does not belong here

1. project-specific runtime decisions (store those in `docs/DECISIONS.md`)
2. active default routing rules without approval
3. unverified social-only claims without primary source links
